# 页面编辑说明

欢迎参与本项目的文档编写！

我们将在本页面中指出对于页面修改的一些注意事项和规范，帮助你更好地进行内容编辑和维护

## 1. 基础语法支持

本项目支持所有标准的 Markdown 语法，您可以在这里找到所有的基础语法教学：
- [Markdown 基础语法](https://markdown.com.cn/basic-syntax/)

---

## 2. 目录结构与内容修改
为了保持文档的清晰和易维护，所有页面按照如下规则进行读取

* 文件夹名称即为页面中`目录`的显示名称，以`.`开头命名的所有文件和目录不会显示在实际页面中
* 建议将图片存放在`images`文件夹中，或者任意一个以`.`开头的目录中，避免造成读取混乱
* 所有需要读取的文件均为`.md`格式
* 提交内容时，请先提交`Pull Request`，并在 PR 描述中注明修改内容和目的，等待审核后合并

---
## 3. 页面中的自定义内容

为了让文档更加生动、重点更加突出，我们在前端渲染时引入了 **7 种不同颜色** 的自定义提示容器

感谢Napcat文档的自定义容器设计，请在Github上支持他们 [Napcat文档](https://github.com/NapNeko/NapCatDocs)

你只需要使用 `:::` 包裹内容，并在第一行写上容器的类型即可。如果需要**自定义标题**，直接写在类型后面加个空格即可。

### 我们提供了如下类型的提示容器：

>可用类型：`info`、`tip`、`warning`、`danger`、`note`、`important`、`caution`

```markdown
::: tip 这里是标题
这是一段示例代码，展示自定义提示容器的使用方法
:::
```
**上述内容的渲染效果如下**

::: tip 这里是标题
这是一段示例代码，展示自定义提示容器的使用方法
:::

**类似的，我们所有的提示容器样式如下：**

::: info 信息
信息内容
:::

::: tip 提示
提示内容
:::

::: warning 警告
警告内容
:::

::: danger 危险
危险内容
:::

::: note 记录
记录内容
:::

::: important 重要
重要内容
:::

::: caution 谨慎
注意内容
:::

## 4. 代码块展示

支持标准的语法高亮格式（在代码块开头标明语言名称）：

```java
// 这是一个 Java 代码块示例
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```
